# API Schemas and Discovery Documents

This directory contains generated OpenAPI schemas and Kubernetes API discovery documents for the oauth-apiserver.

## Directory Structure

- **openapi-spec/**: OpenAPI schema definitions
  - `swagger.json`: OpenAPI v2 schema
  - `v3/`: OpenAPI v3 schemas, one file per API group
- **discovery/**: Kubernetes API discovery documents
  - `aggregated_v2.json`: Aggregated discovery document (v2 format)
  - `apis.json`: API group list
  - `v3-discovery.json`: OpenAPI v3 discovery index
  - Individual API group discovery documents

## Generation

These files are automatically generated by running:

```bash
make update
```

Or directly:

```bash
hack/update-openapi-spec.sh
```

## Verification

CI runs `make verify` which includes `hack/verify-openapi-spec.sh` to ensure these schemas stay in sync with the actual API definitions.

If verification fails, run `make update` to regenerate the schemas.

## Purpose

These schemas serve multiple purposes:

1. **API Documentation**: Provide machine-readable API definitions
2. **Client Generation**: Enable automatic client SDK generation
3. **API Discovery**: Allow clients to discover available APIs dynamically
4. **Schema Validation**: Ensure API stability and prevent breaking changes
5. **CI Gating**: Prevent merging PRs with outdated or inconsistent API schemas
